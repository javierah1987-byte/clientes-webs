Subir webs a Vercel automáticamente

Guía completa — de Claude a URL pública en menos de 2 minutos por web

📥 Descargas el .html de Claude
→
📁 Lo metes en la carpeta
→
🔄 GitHub Desktop sincroniza
→
⚡ Vercel despliega solo
→
🔗 URL lista para enviar
1
Instalar GitHub Desktop
5 minutos · solo una vez
1
Descarga GitHub Desktop
Ve a desktop.github.com y descarga la versión para tu sistema operativo (Windows o Mac). Es gratuito.
2
Crea una cuenta en GitHub (si no tienes)
Ve a github.com → Sign up. Es gratis. Guarda bien el usuario y contraseña.
3
Inicia sesión en GitHub Desktop
Abre la app y haz clic en "Sign in to GitHub.com". Introduce tus credenciales.
2
Crear el repositorio de webs
3 minutos · solo una vez
1
Crea el repositorio en GitHub Desktop
File → New Repository. Ponle el nombre clientes-webs. Elige una carpeta fácil de recordar, como el Escritorio o Documentos. Marca "Initialize this repository with a README". Haz clic en Create Repository.
2
Publica el repositorio en GitHub
En la barra superior verás el botón "Publish repository". Haz clic. Deja la opción "Keep this code private" sin marcar (público es necesario para Vercel gratis). Confirma.
3
Crea una subcarpeta para cada cliente
Dentro de la carpeta clientes-webs/ que se creó en tu ordenador, crea subcarpetas con el nombre del cliente. Ejemplo:
clientes-webs/
├── gb-baterias/
│ └── index.html
├── vr-interiorismo/
│ └── index.html
├── maderas-el-pino/
│ └── index.html
Cada index.html es la web que descargaste de Claude, renombrada a index.html.
💡 Truco: Renombra siempre el archivo de Claude a index.html dentro de su carpeta. Así Vercel lo detecta automáticamente como página principal.
3
Conectar Vercel con GitHub
5 minutos · solo una vez
1
Entra en Vercel
Ve a vercel.com e inicia sesión con tu cuenta de GitHub (botón "Continue with GitHub").
2
Importa el repositorio
Haz clic en "Add New → Project". Verás el repositorio clientes-webs. Haz clic en "Import".
3
Configura el proyecto
En la pantalla de configuración, deja todo por defecto. Haz clic en "Deploy". Vercel desplegará el repositorio completo.
4
Copia la URL de cada carpeta
Cada subcarpeta se convierte en una ruta de la URL. Por ejemplo:
https://clientes-webs.vercel.app/gb-baterias
https://clientes-webs.vercel.app/vr-interiorismo
https://clientes-webs.vercel.app/maderas-el-pino
4
El flujo diario — añadir una nueva web
2 minutos por web
1
Descarga la web de Claude
Cuando Claude te entregue la web, descarga el archivo HTML.
2
Crea la carpeta del cliente
Dentro de clientes-webs/ crea una carpeta con el nombre del cliente (sin espacios, en minúsculas). Mueve el HTML ahí dentro y renómbralo index.html.
3
Sincroniza con GitHub Desktop
Abre GitHub Desktop. Verás el archivo nuevo en la lista de cambios. Escribe un mensaje corto (ej: "Web Clinisalud") en el campo de abajo a la izquierda. Haz clic en "Commit to main" y luego en "Push origin".
4
Vercel despliega automáticamente
En 30–60 segundos, Vercel detecta el cambio en GitHub y despliega la nueva web. La URL ya está activa.
5
Copia la URL y envía el email
La URL es siempre: https://clientes-webs.vercel.app/nombre-carpeta. Pégala en el panel de clientes y envía el email.
🚀

Sistema listo

Con este flujo puedes pasar de "Claude genera la web" a "URL lista para enviar al cliente" en menos de 2 minutos. Para 40 webs semanales, son menos de 90 minutos de trabajo de subida total.

⚠️ Plan gratuito de Vercel: Perfecto para este uso. Sin límite de proyectos, 100 GB de ancho de banda al mes. Con webs estáticas de texto e imágenes externas, ese límite es prácticamente imposible de alcanzar.
